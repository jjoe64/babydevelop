Der Menüpunkt Bearbeiten

  • »Rundschlag«

    Führt alle Schritte zum Erstellen und Ausführen des aktuell geöffneten Projekts durch. D.h. kompilieren aller zum Projekt gehörender Dateien mit anschließendem linken und ausführen. Im Meldungsfenster werden dabei die Kompilermeldungen ausgegeben. In BabyDevelop-typischer Weise wird Ihnen die Entscheidung über Erfolg oder Misserfolg Ihrer Programmierkunst nicht abgenommen. Beantworten Sie also erst dann die Frage zur Durchführung weiterer "Rundschlag"-Schritte, wenn Sie die Ausgaben des Kompiliers gewissenhaft überprüft haben :-)
    Dazu ein Beispiel:
      Sie finden im BabyDevelop-Verzeichnis unter beispiele/mine eine (einfache) Version eines Programms das schon viele ComputerUser erfeute. Öffnen Sie das Projekt mit Projek->Öffnen... und Sie werden feststellen, dass es aus sieben Dateien besteht. Das Kompilieren, Linken und Ausführen vollzieht sich bei diesem Programm auf folgende Weise:

      M i n e n   s u c h e n

      Datei:main.cppmine.cppmine.hspieldaten.cppspieldaten.hspieloberflaeche.cppspieloberflaeche.h
      k o m p i l i e r e n
      erzeugt wird:main.omine.ospieldaten.ospieloberflaeche.o
       
      verändert wird: spieldaten.cpp 
       kompilieren 
      erzeugt wird: spieledaten.o 
       
      verändert wird: spieldaten.h 
       kompilieren 
      erzeugt wird: spieledaten.o 
       
       linken aller objekt-Dateien
      erzeugt wird:babydevelop (Binärdatei)

      Erst eine/die Binärdatei kann dann gestartet werden.

  • »Kompilieren«

    Kompiliert nur die Datei (main.cpp) oder Klasse (h- und cpp-Datei) deren Reiter (Tabulator) aktiviert ist und erzeugt deren Objektdatei. Beachten Sie, dass Veränderungen im Quellcode zuvor gespeichert werden müssen. Bei "Automatisch speichern" auf "Ein" wird dies von BabyDevelop gesteuert (Menü: Bearbeiten, Einstellungen, Automatisch speichern).

  • »Linken«

    Hier werden alle Objektdateien des Projekts gelinkt und die ausführbare Binärdatei erzeugt.

  • »Ausführen«

    Ausführen der aktuell vorhandenen Binärdatei des Projekts.

  • »Alle kompilieren«

    Unter diesem Menüpunkt werden alle zum Projekt gehörenden Dateien kompiliert und die entsprechenden Objektdateien erzeugt.

  • »Neuer Prototyp...«

    Erstellt für ein C-Projekt die h- und c-Datei einer Funktion.

  • »Neue allgemeine Klasse...«

    Erzeugen Sie hiermit allgemeine Klassen die keine Qt-Widgets enthalten. Es werden die h- und cpp-Dateien mit ihren Standard-Konstruktoren und -Destruktoren erzeugt. Hier ein Beispiel für die h-Datei einer Klasse mit dem Namen Beispiel: 
       class Beispiel
   {
   public:
      Beispiel();   /* Konstruktor */
     ~Beispiel();   /* Destruktor  */
   };
    Anmerkung: Sie können nachträglich aus einer "allgemeinen-Klasse" keine "Qt-Klasse" machen. Die von BabyDevelop selbständig erzeugten "kompilier-Scripte" für "allgemeine-Klassen" enthalten nicht den für "Qt-Klassen" notwendigen MetaObjectCompiler (moc).

  • »Neue grafische Klasse...«

    Erzeugen Sie hiermit Klassen die Qt-Klassen enthalten sollen. Dabei werden die h- und cpp-Dateien mit ihren Standard-Konstruktoren und -Destruktoren erzeugt. Auch wird das Q_OBJECT-Makro eingefügt.

    Beachte: BabyDevelop leitet grafische Klassen von der BabyDevelop-eigenen Klasse ProgrammFenster ab. Die Klasse ProgrammFenster ist ihrerseits lediglich von QWidget abgeleitet. Der Vorteil liegt darin, dass dadurch der Standardkonstruktor keine Parameter benötigt. Hier ein Beispiel für die h-Datei einer Klasse mit dem Namen Beispiel: 
       #include "/opt/babyDevelop/programmfenster.h"
   class Beispiel : public ProgrammFenster
   {
      Q_OBJECT
   public:
      Beispiel();   /* Konstruktor */
     ~Beispiel();   /* Destruktor  */
   };
    Und hier noch zum Vergleich die Variante mit direkter Ableitung von QWidget: 
       #include <qwidget.h>
   class Beispiel : public QWidget
   {
      Q_OBJECT
   public:
      Beispiel(QWidget* parent=0, const char *name=0); /* Konstruktor */
     ~Beispiel();                                      /* Destruktor  */
   };
    Anmerkung: Sie können nachträglich aus einer "Qt-Klasse" keine "allgemeine-Klasse" machen, da die von BabyDevelop selbständig erzeugten "kompilier-Scripte" für Qt-Klassen den MetaObjectCompiler (moc) enthalten. Dieser würde beim Kompilieren einer "allgemeinen-Klasse" den Kompiliervorgang mit einer Fehlermeldung abbrechen.

  • »Prototyp importieren...«

    Zum Importieren einer h- und zugehöriger c-Datei einer "ausgelagerten" Funktion. Der Quellcode (h- und c-Datei) wird ins Projektverzeichnis kopiert.

  • »Klasse importieren...«

    Dieser Menüpunkt ist zum Importieren beliebiger Klassen. Der Quellcode (h- und cpp-Datei) wird ins Projektverzeichnis kopiert. Unterschieden wird zwischen zwei Arten von Klassen: Allgemeine Klasse und Grafische Klasse. Vergleiche hierzu auch die Anmerkungen zu "Neue allgemeine Klasse..." und "Neue grafische Klasse...".

  • »Prototyp löschen«

    Löschen des aktiven Prototyps. Es werden unabhängig von aktiver h- oder c-Datei stets beide Dateien gelöscht.

  • »Klasse löschen«

    Löschen der aktiven Klasse. Es werden unabhängig von aktiver h- oder cpp-Datei stets beide Dateien gelöscht.

  • »Einstellungen...«

    Hier können verschiedene Einstellungen zum Optimieren von BabyDevelop vorgenommen werden. Es sind benutzer-/projekt-spezifische Einstellungen die im Homeverzeichnis in der Datei .babydevelop_user.rc (versteckte Datei) bzw. im Projektverzeichnis in der Datei bdpProjekt.rc gespeichert werden.
    (Weiere) Compiler- und Linkeroptionen können hier auch für den GP2X angegeben werden.
Anmerkung: Je nach Projekttyp (C-, C++, Qt/C++- oder GP2X/SDL-Projekt) sind die entsprechenden Menüpunkte (de)aktiviert.
